# How to use this course

> Read this first — it will save you hours.

This course is **self-paced**. There are 50 lessons and 7 projects. You don't have to finish it in any specific number of weeks. The rule is: *understand one lesson well before moving to the next.*

---

## What you need

- A computer.
- A free code editor. Download **Visual Studio Code** from <https://code.visualstudio.com> — it's free and works on Windows, Mac and Linux.
- A web browser (Chrome, Edge, Firefox or Safari — any modern one will do).
- That's it. No installs, no accounts.

---

## How to study each lesson

Every lesson file is a `.md` (Markdown) file. You can read it:

- Directly on GitHub by clicking the file.
- In VS Code by opening the file (it'll render beautifully).
- In any markdown preview tool.

Inside each lesson you will see **HTML and CSS code blocks** like:

```html
<p class="hello">Hello, CSS!</p>
```

```css
.hello {
  color: tomato;
  font-size: 24px;
}
```

When a lesson tells you to **try it yourself**, you do this:

1. Open VS Code.
2. Create a new file (File → New File).
3. Save it as `lesson01.html` in a folder you create called `practice/`.
4. Type or paste the code from the lesson into that file.
5. Save it (`Ctrl+S` or `Cmd+S`).
6. Double-click the file. It opens in your browser. **Refresh the page** every time you change the code to see the update.

> Tip: keep VS Code on one side of your screen and the browser on the other. Change the code → save → switch → refresh → look. Do that 1000 times and you become a CSS developer.

## The rhythm of every lesson

Every lesson has the same 9 sections. Skim the headings before reading and you'll always know where you are:

1. **What you'll learn** — bullet list of goals.
2. **Why this matters** — the practical reason.
3. **The analogy** — a picture that makes the topic stick.
4. **How it works** — short explanation.
5. **Try it yourself** — practical code to copy and edit.
6. **Common mistakes** — what to watch out for.
7. **Debugging tips** — what to check if it doesn't work.
8. **Quick quiz** — 3 questions to test yourself.
9. **Mini challenge** — a small extension to push your skill.
10. **What's next** — link to the next lesson.

You can always come back to a lesson by **browsing** — do not memorize. Concepts return in later lessons.

---

## The 3 study rules

1. **Type the code, don't copy-paste it.** Your fingers learn faster than your eyes.
2. **Break things on purpose.** Change a number, save, refresh — see what happens. Curiosity is the best teacher.
3. **When stuck for more than 10 minutes, ask.** Don't break your flow silently. Write down the question so you remember.

## How to mark progress

Keep a small notebook or a `progress.md` file inside `practice/`. After each lesson write one line:

```
✓ 14 — Borders & rounded corners
```

This gives you a sense of momentum.

---

## When you finish all 50 lessons

You will move to the **projects folder**. Each project has clear specifications, an example solution, and bonus challenges. Finish them in this order:

```
Project 1 → 2 → 3 → 4 → 5 → 6 → 7
```

By project 7 you'll have built a real responsive portfolio website.

Then the final lesson ([Lesson 50: From pure CSS to Tailwind](lessons/50-to-tailwind.md)) maps everything you've learned into Tailwind utility classes so the transition is smooth, not scary.
